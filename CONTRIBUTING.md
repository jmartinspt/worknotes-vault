## Contribution Guidelines

### Template Standards
- **Consistent structure**: Follow existing template patterns
- **Clear documentation**: Include usage instructions and examples
- **Metadata compliance**: Use standard frontmatter format
- **Templater compatibility**: Ensure templates work with Templater plugin

### Documentation Standards
- **Clear explanations**: Write for users of all experience levels
- **Practical examples**: Include real-world usage scenarios
- **Up-to-date information**: Ensure compatibility with current Obsidian version
- **Proper formatting**: Follow Markdown best practices

### Code Quality
- **Test templates**: Verify templates work in clean Obsidian vault
- **Validate queries**: Ensure Dataview queries are efficient and correct
- **Check links**: Verify all internal links resolve properly
- **Follow naming conventions**: Match existing file and folder patterns

## Submission Process

### 1. Fork and Clone
```bash
git clone https://github.com/yourusername/worknotes-vault.git
cd worknotes-vault
```

### 2. Create Feature Branch
```bash
git checkout -b feature/your-feature-name
```

### 3. Make Changes
- Add your improvements
- Test in Obsidian
- Update documentation
- Add examples if needed

### 4. Submit Pull Request
- Clear description of changes
- Reference any related issues
- Include screenshots if visual changes
- Test instructions for reviewers
